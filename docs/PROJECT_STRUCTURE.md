# Project Structure

## Top-level folders
- `calibration/`: Scripts to capture user-specific raw EMG data and generate calibration JSON.
- `matlab_realtime/`: MATLAB runtime pipeline for feature extraction and online prediction calls.
- `model/`: Python training script and exported trained model.
- `server/`: FastAPI inference server and keyboard action layer.
- `docs/`: Repository documentation for setup, usage, and architecture.

## Runtime data and artifacts
- `calibration/calibration_raw.mat`: Generated by calibration recording. Not intended for version control.
- `calibration/user_calibration.json`: Generated user profile mapping. Not intended for version control.
- `model/svm_quadratic_python.joblib`: Trained model artifact used by server at runtime.

## Data contract between MATLAB and Python
- MATLAB sends `POST /predict` with JSON payload:
  - `features`: `list[float]` with dimensionality expected by server scaler.
- Python returns:
  - `gesture`: predicted class ID after optional calibration remap.

## Notes for contributors
- Keep path handling relative to repository root.
- Avoid committing personal calibration files.
- Keep hardware-specific values (for example serial `COM` port) configurable and documented.
